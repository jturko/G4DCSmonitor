# G4DCSmonitor: Dry Cask Storage Monitor Simulation

## Overview
G4DCSmonitor is a Geant4-based application designed to simulate a CASTOR 440/84 dry
storage cask and a collimated CLYC (Cs₂LiYCl₆) detector. The simulation models and
measures the neutron and gamma radiation fields generated by spent nuclear fuel
assemblies, with built-in variance reduction techniques to achieve statistical
precision outside the heavy cask shielding.

The application is built on top of Geant4 (with ROOT 6 as a mandatory dependency)
and supports multithreaded execution via `G4RunManagerFactory`.

## Building
```bash
mkdir build && cd build
cmake ..          # requires Geant4 (ui_all vis_all) and ROOT 6
make -j
```
CMake locates Geant4 and **ROOT 6** (`RIO Net ROOTDataFrame Imt`); the build fails if
ROOT is not version 6. The visualization/run macros in `macros/` are copied into the
build directory automatically.

## Command-Line Interface
```
G4DCSmonitor                                  # interactive (UI) session
G4DCSmonitor run.mac                          # batch, analog tracking
G4DCSmonitor run.mac 17                       # batch, output fileNum = 17
G4DCSmonitor --biasing neutron run.mac        # batch, importance-bias neutrons
G4DCSmonitor --biasing gamma run.mac 17       # batch, importance-bias gammas
G4DCSmonitor run.mac --biasing neutron 17     # arguments are order-insensitive
```
* `--biasing` (alias `-b`) accepts `gamma | neutron | none | off`.
* The first positional argument is the macro; the second (optional) integer is a
  file-number suffix appended to the ROOT output name.
* Launching with no positional argument opens a `G4UIExecutive` interactive session.

> **Important:** importance biasing must be enabled in **two** places that must agree:
> 1. on the command line via `--biasing <gamma|neutron>`, **and**
> 2. in the macro via `/dcs-monitor/det/useBiasing true`.
>
> A mismatch triggers a fatal `BiasingConfigError`. (This is because a parallel world /
> importance store can only be built for a single particle type.)

## Physics List
The simulation uses a custom modular physics list (`PhysicsList.cc`) built from the
following constructors:

* **EM**: `G4EmStandardPhysics_option4`
* **Hadron elastic**: `G4HadronElasticPhysicsHP`
* **Hadron inelastic**: `G4HadronPhysicsQGSP_BIC_HP`
* **Thermal neutrons**: `G4ThermalNeutrons` (sub-4 eV S(α,β) scattering)
* **Ions**: `G4IonElasticPhysics`, `G4IonPhysicsXS`
* **Gamma-nuclear**: custom `GammaNuclearPhysics` (`G4LowEGammaNuclearModel` up to
  200 MeV, `G4CascadeInterface` above) — a `GammaNuclearPhysicsLEND` variant is also
  available in the source
* **Radioactive decay**: `G4RadioactiveDecayPhysics`
* **Biasing (optional)**: `G4ParallelWorldPhysics("BiasingWorld")` +
  `G4ImportanceBiasing` (registered only when `--biasing gamma|neutron` is given)

`G4ParticleHPManager` flags are configured in `G4DCSmonitor.cc`. The current settings
are:

| Flag | Value |
| :--- | :--- |
| `SetSkipMissingIsotopes`     | `true`  |
| `SetDoNotAdjustFinalState`   | `true`  |
| `SetUseOnlyPhotoEvaporation` | `true`  |
| `SetNeglectDoppler`          | `false` |
| `SetProduceFissionFragments` | `false` |
| `SetUseWendtFissionModel`    | `false` |
| `SetUseNRESP71Model`         | `false` |

### Production Cuts
Coarse global cuts are applied for efficiency in the thick shielding:
* `gamma`: 1.0 mm, `e-`: 10.0 mm, `e+`: 10.0 mm, `proton`: 1.0 µm

A dedicated `CLYCRegion` is created around the CLYC scintillator with finer cuts
(0.1 mm for gamma/e-/e+/proton) to preserve detector response fidelity.

## Geometries

### CASTOR 440/84 Cask
The cask geometry is a high-fidelity representation of a CASTOR 440/84 dry storage
container.
* **Components**: Cast iron body, external heat dispersion fins (10 mm thick, 50 mm
  spacing), helium-filled cavity, stainless steel primary lid, 84 VVER-440 hexagonal
  UO₂ fuel assemblies, and 60 polyethylene moderator rods embedded in the cast iron
  wall.
* **Key dimensions**: 4080 mm tall, outer radius 1270 mm (fin tips at 1330 mm), inner
  cavity radius 900 mm, active fuel length 2420 mm.
* **Initialization**: Placed using `/dcs-monitor/det/castor440/add`.

### CLYC Detector
The detector is a collimated CLYC scintillator. It supports highly modular
parameterization.
* **Components**: CLYC crystal, Aluminum casing, Polyethylene (PE/PEHD) collimator and
  rear plug, Lead (Pb) collimator, and a Lithium Fluoride (LiF) collimator liner.
* **Initialization**: Placed using `/dcs-monitor/det/clyc/add` (origin at front face of
  PE plug) or `/dcs-monitor/det/clyc/addByCrystalCenter` (origin at CLYC crystal
  center).

### Custom Materials
In addition to standard NIST materials, the following are defined in code
(`GeometryCLYC::BuildMaterials`, `GeometryCASTOR440::BuildMaterials`):

| Material | Density | Composition |
| :--- | :--- | :--- |
| `CLYC`          | 3.31 g/cm³  | Cs₂ + ⁶Li-enriched Li + Y + Cl₆ |
| `LiF`           | 2.635 g/cm³ | ⁶Li-enriched Li + F (95% ⁶Li / 5% ⁷Li) |
| `PEHD`          | 0.96 g/cm³  | C + H (thermal-scattering H, `H_POLYETHYLENE`) |
| `PEHD_borated`  | 1.01 g/cm³  | 95% PEHD + 5% B |
| `CastIron`      | 7.3 g/cm³   | 96% Fe / 4% C |
| `UO2`           | 10.5 g/cm³  | U + 2×O |

### Geometry Macro Commands
Volumes are placed at the currently defined position and rotation state.

| Command | Description | Default Unit |
| :--- | :--- | :--- |
| `/dcs-monitor/det/setPosition [x] [y] [z]` | Set position for the next volume. | `mm` |
| `/dcs-monitor/det/setRotation [x] [y] [z]` | Set rotation for the next volume. | `deg` |
| `/dcs-monitor/det/castor440/add` | Instantiate a CASTOR 440 cask. | N/A |
| `/dcs-monitor/det/clyc/add` | Instantiate a CLYC detector (origin at front face). | N/A |
| `/dcs-monitor/det/clyc/addByCrystalCenter` | Instantiate a CLYC detector (origin at crystal center). | N/A |

**(Command names verified against `DetectorMessenger.cc`.)**

| :--- | :--- | :--- |

| **Pb Collimator** | `setPbColInnerRadius`, `setPbColOuterRadius`, `setPbColLength` | `setPbColMaterial` |

## Primary Generator Modes
The particle source distribution mode is controlled via
`/dcs-monitor/gun/sourceMode [mode]`.

1.  **`GPS`**: Standard Geant4 General Particle Source (`/gps/` commands).
2.  **`CASTOR440_surface`**: Cosine-weighted (Lambertian) outward flux from the external
    surface (top, bottom, and cylindrical side, weighted by area) of a specified cask.
    * Requires: `/dcs-monitor/gun/caskNum [id]`.
3.  **`CASTOR440_fuel`**: Uniform-volume, isotropic emission from a specific fuel
    assembly inside a specific cask.
    * Requires: `/dcs-monitor/gun/caskNum [id]` and `/dcs-monitor/gun/fuelNum [id]`
      (0 to 83).
4.  **`CASTOR440_fuel_biased`**: Same spatial origin as `CASTOR440_fuel`, but emission
    direction is bounded to a cone intersecting the CLYC bounding sphere; the primary
    vertex weight is adjusted to compensate.
    * Additional control: `/dcs-monitor/gun/clycBoundingRadius [r] [unit]` (default
      150 mm).
5.  **`CASTOR440_surface_from_TTree`**: Phase-space *replay* mode. Resamples crossings
    from a `surfaceFlux` ROOT tree produced by a previous run, transports them out of
    the cask, and into the detector. Configured via the `/dcs-monitor/surf/...` driver
    (see below). This is the engine behind the two-part workflow in
    `examples/p1-surface_flux_generation` and `examples/p2-detector_response`.

### Watt Fission-Spectrum Source
For neutron sources, the `CASTOR440_fuel*` modes can sample energies from a Watt
spectrum (Cashwell–Everett rejection algorithm):

| Command | Description | Default |
| :--- | :--- | :--- |
| `/dcs-monitor/gun/useWattSpectrum [bool]` | Enable Watt sampling (forces `neutron`). | off |
| `/dcs-monitor/gun/wattA [E]`              | Watt parameter `a`. | 0.988 MeV |
| `/dcs-monitor/gun/wattB [val]`            | Watt parameter `b` (1/MeV). | 2.249 /MeV |

### Surface-Flux Replay Driver (`/dcs-monitor/surf/...`)
Used with `sourceMode CASTOR440_surface_from_TTree`. The sampler (`SurfaceFluxSampler`,
a master-side singleton) loads the tree once, builds a Walker alias table, and lets
worker threads draw crossings in O(1).

| Command | Description |
| :--- | :--- |
| `sourceFile [path]`            | Path to the step-1 ROOT file (`surfaceFlux` tree). |
| `maxEntriesLoadedFromTree [n]` | Cap entries read at load time (`<=0` = all). |
| `sourcePid [pid]`              | PDG filter: `0`=any, `2112`=n, `22`=γ. |
| `smearPhi [rad]`, `smearZ [len]`, `smearAngle [rad]`, `smearEfrac [frac]` | Resampling smear widths. |
| `caskNum [c]`                  | Cask whose geometry/placement to replay into. |
| `sourceRotZ [deg]`            | Rotate the source about +z (C6 hexant reuse). |
| `numPrimaries [n]`            | Number of primaries that produced the input tree. |
| `decayRate [1/s]`            | Activity of the source. |
| `startRun [t] [unit]`        | Compute N primaries from rate × time × (kept weight / numPrimaries) and launch `/run/beamOn`. |

## Variance Reduction (Biasing)

1.  **Geometry Importance Biasing (Weight Windows)** — `GeometryParallelBiasing`
    * **Commands (pre-init only)**:
      * `/dcs-monitor/det/useBiasing [true/false]`
      * `/dcs-monitor/det/setBiasingShells [N]` (max importance = 2ᴺ)
      * `/dcs-monitor/det/setBiasingInnerRadius [r]` (default 900 mm)
      * `/dcs-monitor/det/setBiasingOuterRadius [r]` (default 1270 mm)
      * `/dcs-monitor/det/setBiasingInnerHeight [h]` (default 3500 mm)
    * **Mechanism**: A parallel `BiasingWorld` overlays the mass geometry. A core
      (importance 1) covering the fuel region is surrounded by nested cylindrical
      shells of uniform radial *and* axial thickness, each doubling the importance of
      its parent. Tracks propagating outward into higher importance are split with
      proportionally reduced weights. Shells are built per cask, so multi-cask setups
      are supported. **Must be paired with the `--biasing` CLI flag** (see CLI section).

2.  **Primary Generator Directional Biasing** — `sourceMode CASTOR440_fuel_biased`
    * Primaries are emitted into a cone toward a (uniformly chosen) CLYC bounding
      sphere. Compensating weight: `weight = N_CLYC × (1 − cosθ_max) / 2`.

3.  **Kinematic Cutoffs (Stepping Action)** — *currently disabled*
    * The `SteppingAction.cc` block that terminates e⁻/e⁺ in the shielding and gammas
      below 50 keV is **commented out** in the current source. Re-enable it there if you
      need this optimization. (The surface-flux tracker in `SteppingAction` *is* active.)

## Output
Data is written to a ROOT file set by `/analysis/setFileName [name]` (a numeric suffix
is appended if a file number was passed on the CLI). The file contains three ntuples:

* **`primary`**: Initial state of generated primaries. Toggled via
  `/dcs-monitor/run/writePrimary [true/false]`. Columns: `pid`, `ekin`, `t`,
  `x`,`y`,`z`, `px`,`py`,`pz`.
* **`hits`**: Always active. CLYC sensitive-detector interactions, aggregated per track
  using a shower-container scheme (secondaries produced inside the crystal are folded
  into their parent's hit). Columns: `pid`, `edep`, `t`, `x`,`y`,`z`, `det`, `weight`.
* **`surfaceFlux`**: Phase-space tracker for gammas/neutrons leaving the cask body
  (into the world or fins), recorded in **cask-local coordinates** for downstream
  replay. Toggled via `/dcs-monitor/run/writeSurfaceFlux [true/false]`. Columns: `pid`,
  `ekin`, `t`, `x`,`y`,`z`, `px`,`py`,`pz`, `weight`, `evtNb`.

Helper ROOT macro `scripts/analyze_primary.C` provides a quick visual inspection of the
`primary` ntuple; `scripts/PlotWattSpectrum.C` plots a Watt spectrum.

## Runtime Features
* **Multithreading**: `/run/numberOfThreads [N]`; ntuples auto-merged across threads.
* **Progress bar**: monotonic-clock `ProgressBar` printing instantaneous/averaged event
  rate, elapsed/remaining time, and a wall-clock ETA.
* **Seeded randomness**: engine seeded with `time(NULL)` at startup; status printed.
* **Interactive / batch modes**: see the CLI section.

## Examples
The `examples/` directory contains validated workflows:
* **`geometric_biasing/`** — validates the directional fuel bias vs. an unbiased run.
* **`p1-surface_flux_generation/`** — step 1: generate cask surface flux per source.
* **`p2-detector_response/`** — step 2: replay surface flux into the CLYC detector,
  exploiting the basket's C6 symmetry. Includes analysis macros.
