# G4DCSmonitor: Dry Cask Storage Monitor Simulation

## Overview
G4DCSmonitor is a Geant4-based application designed to simulate a CASTOR 440/84 dry storage cask and a collimated CLYC (Cs₂LiYCl₆) detector. The simulation is used to model and measure the neutron and gamma radiation fields generated by spent nuclear fuel assemblies.

## Geometries

### CASTOR 440/84 Cask
The cask geometry is a high-fidelity representation of a CASTOR 440/84 dry storage container.
* **Components**: Cast iron body, external heat dispersion fins, helium-filled cavity, stainless steel primary lid, 84 VVER-440 hexagonal UO₂ fuel assemblies, and 60 polyethylene moderator rods drilled into the cast iron wall.
* **Initialization**: Placed using `/dcs-monitor/det/castor440/add`.

### CLYC Detector
The detector is a collimated CLYC scintillator. It supports highly modular parameterization.
* **Components**: CLYC crystal, Aluminum casing, Polyethylene (PE/PEHD) collimators and rear plug, Lead (Pb) collimator, and a Lithium Fluoride (LiF) collimator liner.
* **Initialization**: Placed using `/dcs-monitor/det/clyc/add` or `/dcs-monitor/det/clyc/addByCrystalCenter`.

### Geometry Macro Commands
Volumes are placed at the currently defined position and rotation state.

| Command | Description | Default Unit |
| :--- | :--- | :--- |
| `/dcs-monitor/det/setPosition [x] [y] [z]` | Set position for the next volume. | `mm` |
| `/dcs-monitor/det/setRotation [x] [y] [z]` | Set rotation for the next volume. | `deg` |
| `/dcs-monitor/det/castor440/add` | Instantiate a CASTOR 440 cask. | N/A |
| `/dcs-monitor/det/clyc/add` | Instantiate a CLYC detector (origin at front face). | N/A |
| `/dcs-monitor/det/clyc/addByCrystalCenter` | Instantiate a CLYC detector (origin at crystal center). | N/A |

## Primary Generator Modes
The particle source distribution mode is controlled via `/dcs-monitor/gun/sourceMode [mode]`. 

1.  **`GPS`**: Utilizes the standard Geant4 General Particle Source (`/gps/` commands).
2.  **`CASTOR440_surface`**: Generates an isotropic flux from the external surface area of the CASTOR body (top, bottom, and cylindrical sides).
3.  **`CASTOR440_fuel`**: Generates a uniform volume flux from a specific fuel assembly inside a specific cask.
    * Requires: `/dcs-monitor/gun/caskNum [id]` and `/dcs-monitor/gun/fuelNum [id]` (0 to 83).
    * Particle type and energy are set using standard `/gun/` commands.
4.  **`CASTOR440_fuel_biased`**: Identical spatial origin to `CASTOR440_fuel`, but the emission direction is strictly bounded to a cone intersecting the CLYC detector bounding sphere. 

## Variance Reduction (Biasing)
To achieve adequate statistical precision outside the thick cask shielding, three variance reduction techniques are implemented.

1.  **Geometry Importance Biasing (Weight Windows)**
    * **Command**: `/dcs-monitor/det/useBiasing [true/false]` (Must be called before `/run/initialize`).
    * **Mechanism**: A parallel world geometry overlays the mass geometry. The inner core covering the fuel region is assigned a base importance of 1. Surrounding concentric cylindrical shells increase in importance outward, multiplying by factors of 2 for each shell. As particle tracks propagate outwards through the cask shielding into regions of higher importance, they are systematically split. To conserve physical accuracy, the statistical weight of these split tracks is proportionally reduced. This artificially inflates the population of tracks reaching the outer shell and the detector, yielding viable counting statistics despite heavy attenuation. 
2.  **Primary Generator Directional Biasing**
    * **Command**: `/dcs-monitor/gun/sourceMode CASTOR440_fuel_biased`
    * **Mechanism**: This run mode applies geometric biasing at the source level. Primary particles originating in the fuel are preferentially emitted within a constrained solid angle (a cone directed specifically towards the CLYC detector). A fractional statistical weight is applied to the primary vertices to compensate for the artificially increased emission probability in the detector's direction.
3.  **Kinematic Cutoffs (Stepping Action Termination)**
    * **Mechanism**: Executed in `SteppingAction.cc`. If a particle is inside the cask shielding (`CastorBody`, `Cavity`, or `FuelPhys`), all electrons and positrons are immediately terminated. Gammas with kinetic energy below 200 keV are also terminated to suppress the computation of the low-energy Compton continuum and optimize overall computational efficiency.

## Output
Data is output to a ROOT file specified by `/analysis/setFileName [name]`. The file contains two Ntuples:

* **`primary`**: Logs the initial state of generated primary particles. This tree is toggled using the `/dcs-monitor/run/writePrimary [true/false]` command. When enabled, it records the particle ID (`pid`), kinetic energy (`ekin`), global time (`t`), origin coordinates (`x`, `y`, `z`), and the momentum direction vector (`px`, `py`, `pz`) for each generated primary.
* **`hits`**: Logs sensitive detector interactions in the CLYC crystal. Tracks `pid`, `edep` (energy deposited), `t` (global time), `x`, `y`, `z`, `det` (detector copy number), and `weight` (statistical weight from biasing).

## Example Run Macro
```macro
# Enable multithreading
/run/numberOfThreads 4

# Enable Geometry Importance Biasing
/dcs-monitor/det/useBiasing true

# Configure and place CLYC Detector
/dcs-monitor/det/setPosition 0 0 -1400 mm
/dcs-monitor/det/setRotation 0 0 0
/dcs-monitor/det/clyc/add

/dcs-monitor/det/clyc/setCrystalMaterial CLYC
/dcs-monitor/det/clyc/setCrystalRadius 12.5 mm
/dcs-monitor/det/clyc/setCrystalLength 25.0 mm
/dcs-monitor/det/clyc/setPbColLength 50.0 mm
/dcs-monitor/det/clyc/setPEHDColLength 50.0 mm

# Place CASTOR 440/84 Cask
/dcs-monitor/det/setPosition 0 0 0 m
/dcs-monitor/det/setRotation -90 0 0
/dcs-monitor/det/castor440/add

# Initialize Geometry and Physics
/run/initialize

# Set Generator Mode to Biased Fuel Source
/dcs-monitor/gun/sourceMode CASTOR440_fuel_biased
/dcs-monitor/gun/caskNum 0
/dcs-monitor/gun/fuelNum 20
/gun/particle gamma
/gun/energy 1.332 MeV

# Configure Output
/analysis/setFileName biased_run_data
/dcs-monitor/run/writePrimary false

# Execute Run
/run/beamOn 1000000
