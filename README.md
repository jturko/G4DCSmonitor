# G4DCSmonitor: Dry Cask Storage Monitor Simulation

## Overview
G4DCSmonitor is a Geant4-based application designed to simulate a CASTOR 440/84 dry storage cask and a collimated CLYC (Cs₂LiYCl₆) detector. The simulation models and measures the neutron and gamma radiation fields generated by spent nuclear fuel assemblies, with built-in variance reduction techniques to achieve statistical precision outside the heavy cask shielding.

The application is built on top of Geant4 (with ROOT 6 as a mandatory dependency) and supports multithreaded execution via `G4RunManagerFactory`.

## Physics List
The simulation uses a custom modular physics list (`PhysicsList.cc`) built from the following constructors:

* **EM**: `G4EmStandardPhysics_option4`
* **Hadron elastic**: `G4HadronElasticPhysicsHP`
* **Hadron inelastic**: `G4HadronPhysicsQGSP_BIC_HP`
* **Thermal neutrons**: `G4ThermalNeutrons` (sub-4 eV S(α,β) scattering)
* **Ions**: `G4IonElasticPhysics`, `G4IonPhysicsXS`
* **Gamma-nuclear**: custom `GammaNuclearPhysics` (`G4LowEGammaNuclearModel` up to 200 MeV, `G4CascadeInterface` above)
* **Radioactive decay**: `G4RadioactiveDecayPhysics`
* **Biasing (optional)**: `G4ParallelWorldPhysics` + `G4ImportanceBiasing` for gammas and neutrons

`G4ParticleHPManager` flags are configured in `G4DCSmonitor.cc` (e.g. `ProduceFissionFragments = true`, `UseOnlyPhotoEvaporation = true`, `SkipMissingIsotopes = false`).

### Production Cuts
Coarse global cuts are applied for efficiency in the thick shielding:
* `gamma`: 1.0 mm, `e-/e+`: 10.0 mm, `proton`: 1.0 µm

A dedicated `CLYCRegion` is created around the CLYC scintillator with finer cuts (0.1 mm for gamma/e-/e+/proton) to preserve detector response fidelity.

## Geometries

### CASTOR 440/84 Cask
The cask geometry is a high-fidelity representation of a CASTOR 440/84 dry storage container.
* **Components**: Cast iron body, external heat dispersion fins (10 mm thick, 50 mm spacing), helium-filled cavity, stainless steel primary lid, 84 VVER-440 hexagonal UO₂ fuel assemblies, and 60 polyethylene moderator rods embedded in the cast iron wall.
* **Key dimensions**: 4080 mm tall, outer radius 1270 mm (fin tips at 1330 mm), inner cavity radius 900 mm.
* **Initialization**: Placed using `/dcs-monitor/det/castor440/add`.

### CLYC Detector
The detector is a collimated CLYC scintillator. It supports highly modular parameterization.
* **Components**: CLYC crystal, Aluminum casing, Polyethylene (PE/PEHD) collimators and rear plug, Lead (Pb) collimator, and a Lithium Fluoride (LiF) collimator liner.
* **Initialization**: Placed using `/dcs-monitor/det/clyc/add` (origin at front face of PE plug) or `/dcs-monitor/det/clyc/addByCrystalCenter` (origin at CLYC crystal center).

### Custom Materials
In addition to standard NIST materials, the following are defined in code:

| Material | Density | Composition |
| :--- | :--- | :--- |
| `CLYC` | 3.31 g/cm³ | Cs₂ + ⁶Li-enriched Li + Y + Cl₆ |
| `LiF`  | 2.635 g/cm³ | ⁶Li-enriched Li + F (95% ⁶Li / 5% ⁷Li) |
| `CastIron` | 7.3 g/cm³ | 96% Fe / 4% C |
| `UO2`  | 10.5 g/cm³ | U + 2×O |

## Primary Generator Modes
The particle source distribution mode is controlled via `/dcs-monitor/gun/sourceMode [mode]`. 

1.  **`GPS`**: Utilizes the standard Geant4 General Particle Source (`/gps/` commands).
2.  **`CASTOR440_surface`**: Generates a cosine-weighted (Lambertian) outward flux from the external surface area of a specified CASTOR body (top, bottom, and cylindrical sides, weighted by area).
    * Requires: `/dcs-monitor/gun/caskNum [id]`.
    * Particle type and energy are set using standard `/gun/` commands.
3.  **`CASTOR440_fuel`**: Generates a uniform volume flux with isotropic emission from a specific fuel assembly inside a specific cask.
    * Requires: `/dcs-monitor/gun/caskNum [id]` and `/dcs-monitor/gun/fuelNum [id]` (0 to 83).
    * Particle type and energy are set using standard `/gun/` commands.
4.  **`CASTOR440_fuel_biased`**: Identical spatial origin to `CASTOR440_fuel`, but the emission direction is strictly bounded to a cone intersecting the CLYC detector bounding sphere. The primary vertex weight is automatically adjusted to compensate.
    * Additional control: `/dcs-monitor/gun/clycBoundingRadius [r] [unit]` sets the CLYC bounding sphere radius used by the directional bias (default 150 mm).

## Variance Reduction (Biasing)
To achieve adequate statistical precision outside the thick cask shielding, three variance reduction techniques are implemented.

1.  **Geometry Importance Biasing (Weight Windows)**
    * **Commands (pre-init only)**:
      * `/dcs-monitor/det/useBiasing [true/false]` — enable/disable (default: off)
      * `/dcs-monitor/det/setBiasingShells [N]` — number of shells (max importance = 2ᴺ)
      * `/dcs-monitor/det/setBiasingInnerRadius [r]` — inner core radius (default 900 mm)
      * `/dcs-monitor/det/setBiasingOuterRadius [r]` — outermost shell radius (default 1270 mm)
      * `/dcs-monitor/det/setBiasingInnerHeight [h]` — full axial height of core (default 3500 mm)
    * **Mechanism**: A parallel world (`BiasingWorld`) overlays the mass geometry. The inner core covering the fuel region is assigned a base importance of 1. Concentric nested cylindrical shells (uniform radial and axial thickness per shell) surround the core, each doubling the importance of its parent. As gamma/neutron tracks propagate outwards through the cask shielding into regions of higher importance, they are systematically split with proportionally reduced statistical weights, yielding viable counting statistics despite heavy attenuation. Shells are instantiated per CASTOR 440 cask, so multi-cask setups are fully supported.

2.  **Primary Generator Directional Biasing**
    * **Command**: `/dcs-monitor/gun/sourceMode CASTOR440_fuel_biased`
    * **Mechanism**: Primary particles originating in the fuel are preferentially emitted within a constrained solid angle (a cone directed towards the CLYC detector's bounding sphere). If multiple CLYC detectors are present, one is chosen uniformly per primary. A fractional statistical weight `weight = N_CLYC × (1 − cosθ_max) / 2` is applied to the primary vertex to compensate for the artificially increased emission probability in the detector's direction.

3.  **Kinematic Cutoffs (Stepping Action Termination)**
    * **Mechanism**: Executed in `SteppingAction.cc`. If a particle is inside the cask shielding (`CastorBody`, `Cavity`, or `FuelPhys`), all electrons and positrons are immediately terminated. Gammas with kinetic energy below **50 keV** are also terminated to suppress computation of the low-energy Compton continuum and optimize computational efficiency.

## Output
Data is output to a ROOT file specified by `/analysis/setFileName [name]`. The file contains three Ntuples, any of which can be toggled via `/dcs-monitor/run/` commands:

* **`primary`**: Logs the initial state of generated primary particles. Toggled via `/dcs-monitor/run/writePrimary [true/false]`. When enabled, it records the particle ID (`pid`), kinetic energy (`ekin`), global time (`t`), origin coordinates (`x`, `y`, `z`), and the momentum direction vector (`px`, `py`, `pz`) for each generated primary.
* **`hits`**: Always active. Logs sensitive detector interactions in the CLYC crystal, aggregated per track using a shower-container scheme (secondaries produced inside the crystal are folded into their parent's hit). Records `pid`, `edep` (total shower energy deposited), `t` (time of first energy deposit), `x`, `y`, `z`, `det` (detector copy number), and `weight` (statistical weight from biasing).
* **`surfaceFlux`**: Phase-space tracker for particles leaving the outer surface of any CASTOR 440 cask (crossings from the cask body or fins into the world volume). Toggled via `/dcs-monitor/run/writeSurfaceFlux [true/false]`. Gammas and neutrons only. Records `pid`, `ekin`, `t`, position (`x`, `y`, `z`), momentum (`px`, `py`, `pz`), statistical `weight`, and `evtNb` (event number) — suitable as input for downstream phase-space replay.

A helper ROOT macro `scripts/analyze_primary.C` is provided for quick visual inspection of the `primary` ntuple (position/momentum/ekin distributions).

## Runtime Features
* **Multithreading**: Controlled via `/run/numberOfThreads [N]`. Ntuples are automatically merged across threads.
* **Progress bar**: A monotonic-clock-based progress bar (`ProgressBar.hh`) prints an updating status line including instantaneous/averaged event rate, elapsed time, remaining time, and ETA wall-clock timestamp.
* **Seeded randomness**: The random engine is seeded with `time(NULL)` at startup; the engine status is printed at run start.
* **Interactive / batch modes**: Launching without arguments opens a `G4UIExecutive` session; otherwise the first argument is executed as a macro.

## Example Run Macro
```macro
# Enable multithreading
/run/numberOfThreads 4

# Enable Geometry Importance Biasing
/dcs-monitor/det/useBiasing true
/dcs-monitor/det/setBiasingShells 10

# Configure and place CLYC Detector
/dcs-monitor/det/setPosition 0 0 -1400 mm
/dcs-monitor/det/setRotation 0 0 0
/dcs-monitor/det/clyc/add

/dcs-monitor/det/clyc/setCrystalMaterial CLYC
/dcs-monitor/det/clyc/setCrystalRadius 12.5 mm
/dcs-monitor/det/clyc/setCrystalLength 25.0 mm
/dcs-monitor/det/clyc/setPbColLength 50.0 mm
/dcs-monitor/det/clyc/setPEHDColLength 50.0 mm

# Place CASTOR 440/84 Cask
/dcs-monitor/det/setPosition 0 0 0 m
/dcs-monitor/det/setRotation -90 0 0
/dcs-monitor/det/castor440/add

# Initialize Geometry and Physics
/run/initialize

# Set Generator Mode to Biased Fuel Source
/dcs-monitor/gun/sourceMode CASTOR440_fuel_biased
/dcs-monitor/gun/clycBoundingRadius 100 cm
/dcs-monitor/gun/caskNum 0
/dcs-monitor/gun/fuelNum 20
/gun/particle gamma
/gun/energy 1.332 MeV

# Configure Output
/analysis/setFileName biased_run_data
/dcs-monitor/run/writePrimary false
/dcs-monitor/run/writeSurfaceFlux false

# Execute Run
/run/beamOn 1000000
